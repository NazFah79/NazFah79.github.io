What is ALARM?

ALARM stands for A Learning and Responding Matrix. It is a tool that groups NESA's key words (directive verbs) by the depth of thinking and response they require — from simple recall all the way to critical evaluation.

💡 The key insight: ALARM is not 5 separate named levels — it's a matrix or spectrum. Key words on the left require short, factual answers. Key words on the right demand longer, higher-order responses with evidence, reasoning, and judgement. The mark allocation of a question usually signals where on the matrix it sits.
How to use ALARM in an exam
1. Spot the directive verb in the question (e.g. "Explain", "Evaluate")
2. Locate it on the ALARM matrix — how deep does your answer need to go?
3. Check the mark allocation — more marks = more depth needed
4. Plan your response using VSCC structure (Verb, Subject, Criteria, Content)
5. Write at the correct depth — don't just describe when asked to evaluate!
⚠️ Common mistake: Students who write "describe" answers for "evaluate" questions miss marks even when their facts are correct. ALARM helps you avoid this.

VSCC Breakdown

Before writing, decode the question using VSCC: identify the Verb, Subject, Criteria, and plan your Content. This ensures you answer exactly what is asked.

What is VSCC?
V
Verb
The directive word — tells you how to answer (e.g. Explain, Evaluate)
S
Subject
The topic the question is about (e.g. cloud computing, firewalls)
C
Criteria
Any specific conditions or focus (e.g. "in terms of cost", "for SMEs")
C
Content
The knowledge, examples and evidence you'll include

ALARM Quick Reference — Mark & Depth Guide
1–2 marks Identify / Define / List / Recall / Name — one or two sentences, factual only
2–4 marks Describe / Outline / Summarise / Classify — key features with some detail
4–6 marks Explain / Compare / Contrast / Apply — cause & effect, reason why, examples required
6–8 marks Analyse / Examine / Discuss / Interpret — components + relationships + implications
8+ marks Evaluate / Assess / Justify / Recommend — judgement with supported criteria, pros/cons
Extended Critically Analyse / Critically Evaluate — all of the above + challenge assumptions, limitations
Pre-Write VSCC Checklist
I have identified the directive verb and located it on the ALARM
I know the subject — exactly what the question is about
I have noted any criteria/conditions in the question
I have planned the content — key points, examples, evidence
My response depth matches the mark allocation
I have not merely described when a higher-order verb was used

How to create a Fine-Grained PAT (step by step)
  1. Go to github.com → Settings → Developer settings → Personal access tokens → Fine-grained tokens
  2. Click Generate new token
  3. Give it a name (e.g. ALARM Writing Coach) and set an expiry
  4. Set Repository access to Only select repositories → choose your repo
  5. Under Repository permissions, set Contents → Read and write
  6. Scroll down and click Generate token — copy it immediately (only shown once!)
  7. Paste it into the PAT field above and click Save Config
